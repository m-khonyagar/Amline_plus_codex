# Contributing

## Workflow
1. Branch from `main`.
2. Keep backend, frontend, and infrastructure changes clearly scoped.
3. Update docs when setup or runtime expectations change.
4. Run the relevant CI-equivalent checks before opening a pull request.

## Expected checks
- backend tests and migrations
- frontend install and build
- `docker compose config`

## Pull requests
Document what changed, how it was validated, and whether any environment variables or deployment expectations changed.
